# AI 知识库 V1 未完成项追踪

## 文档目的

这份文档用于对齐当前项目与“优化文档”之间的差距，避免后续开发只靠聊天记录推进。

适用场景：

- 继续按 V1 目标补齐产品能力
- 每轮开发前确认优先级
- 每轮开发后回填进度
- 作为后续任务拆分和验收依据

---

## 当前判断口径

状态说明：

- `已完成`：主链路可用，用户已能直接使用
- `部分完成`：已有基础能力，但体验、治理或扩展性不足
- `未完成`：文档目标尚未真正落地

当前评估时间：

- 2026-04-16

---

## 一、已完成能力

### 1. 会话上下文与 Memory 基础链路

- 会话已从单纯消息容器升级为状态容器
- 已支持 `sessionSummary` 与 `sessionFacts`
- 聊天上下文已接入“最近消息 + 会话记忆 + 检索结果”
- 会话标题已支持模型生成，不再只取首条用户消息
- 前端已有会话记忆查看与编辑入口

### 2. 文档上传后的异步总结链路

- 文档上传后支持异步处理
- 已支持基础状态：`UPLOADED`、`PROCESSING`、`SUMMARIZED`、`FAILED`
- 已支持更细的处理阶段：`PARSING`、`SUMMARIZING`、`INDEXING`、`COMPLETED`
- 原文、解析文本、总结文本已分层存储
- 总结内容已进入检索层
- 原文 chunk 和 summary chunk 已分层建立索引

### 3. AI 问答主链路

- 已支持 query rewrite
- 已支持检索优先、知识不足时回退到通用回答
- 已支持 `<think></think>` 提取
- `thinking` 已单独存储并在前端折叠展示
- 已支持会话导出 Markdown / PDF
- 已支持前端“停止生成”按钮

### 4. 文档工作台基础能力

- 已支持 Markdown / 文本类文档预览
- 已支持解析文本编辑
- 已支持总结内容独立查看与编辑
- 已支持 AI 自动总结 / 启发式摘要 / 人工编辑摘要切换
- 已支持失败重试
- 已支持失败原因展示

### 5. 基础产品体验

- 聊天页已有本地缓存
- 从 `/chat` 切换到其他模块时，未完成问答可继续在后台执行
- 已接入项目内置本地 embedding 轻量模型
- 已集成 MiniMax provider
- 个人资料基础版已可用，包括头像与简介

---

## 二、部分完成能力

### 1. 文档任务治理

当前进度：

- 已有异步处理
- 已有失败重试
- 已有失败原因
- 已有处理阶段展示

仍然缺少：

- 独立任务中心页面
- 历史任务列表
- 批量查看任务状态
- 任务级取消能力
- 更清晰的任务审计记录

判断：

- 已具备基础可用性，但还不是完整的任务治理系统

### 2. 检索结果可解释性

当前进度：

- 聊天消息下方已显示引用来源
- 已展示文档标题、命中片段、相关度
- 已新增聊天页“检索调试”抽屉
- 已支持展示 query rewrite 前后对比
- 已支持展示 summary/raw 命中类型、chunk 索引、topK 调试结果

仍然缺少：

- 更独立的检索调试页面
- 更丰富的召回对比指标
- 面向非研发用户的检索解释视图

判断：

- 研发调试已基本可用，但还没有完全产品化

### 3. 停止生成

当前进度：

- 前端可取消请求

仍然缺少：

- 后端完整的任务中断与资源回收机制

判断：

- 用户侧可用，系统侧治理未完成

### 4. 会话导出

当前进度：

- 已支持 Markdown / PDF 导出

仍然缺少：

- 更自然的入口位置
- 导出模板配置
- 更完整的可选项控制

判断：

- 功能可用，产品化程度一般

### 5. 文档预览体验

当前进度：

- Markdown 渲染正常
- 文本类文档预览可用
- 总结资产查看区域已独立

仍然缺少：

- 更完整的 PDF 阅读器体验
- 原始二进制文档级别的高保真预览

判断：

- 足够支持当前 V1 使用，但仍有提升空间

### 6. 个人资料功能

当前进度：

- 头像与简介已支持保存

仍然缺少：

- 语言偏好
- 主题偏好
- AI 偏好设置
- 更完整的个人资料面板

判断：

- 仍属于基础版

## 五、建议的下一步开发顺序

### Step 1：文档任务中心

目标：

- 把“异步处理”升级成“可管理任务”

最小交付建议：

- 文档任务列表页
- 按文档查看最近任务
- 显示状态、阶段、开始时间、结束时间、错误信息
- 失败重试
- 处理中取消

### Step 2：检索调试面板

目标：

- 让研发和测试可以直接看到向量检索到底召回了什么

最小交付建议：

- 输入 query
- 展示 rewrite 结果
- 展示 topK chunk
- 显示 chunkType、score、documentTitle、content 预览

### Step 3：后端任务取消机制

目标：

- 让“停止生成 / 取消处理”不只是前端停止等待，而是真正停止后台任务

## 六、文档维护方式

后续每轮开发建议按下面方式更新本文档：

1. 完成一个功能后，把对应项从“未完成”移动到“部分完成”或“已完成”
2. 如果只是做了基础版，优先标注为“部分完成”
3. 每轮开发结束时，补一段“本轮变更”
